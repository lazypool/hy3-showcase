def quick_sort(arr):
    if len(arr) <= 1:
        return arr
    
    pivot = arr[len(arr) // 2]  # 选择中间元素作为基准
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    
    return quick_sort(left) + middle + quick_sort(right)


# 示例
if __name__ == "__main__":

**MoE（Mixture of Experts，混合专家）架构**是一类神经网络设计思路，核心思想是：**模型由多个“专家”子网络组成，每次推理只激活其中一部分，而不是动用全部参数**。这样可以在不显著增加计算量的情况下，大幅扩大模型总参数量，从而提升模型能力。

---

## 一、基本组成
一个典型的 MoE 层通常包含：
1. **多个专家（Experts）**  
   通常是前馈神经网络（FFN），各自负责不同输入模式或知识领域。
2. **门控网络（Router / Gate）**  
   根据输入决定“哪些专家被激活”，以及“各自权重是多少”。
3. **稀疏激活机制**  
   每个 token 一般只路由到 **Top-k 个专家**（如 Top-1、Top-2）。

---

## 二、核心原理（以 Transformer 为例）
在标准 Transformer 中：
- 每个 token 经过一个 FFN
- 在 MoE 中：
  - 原 FFN 被替换为 **N 个专家 FFN**
  - 门控网络计算 token 对各专家的得分
  - 只选分数最高的 k 个专家参与计算
  - 输出按门控权重加权求和

👉 结果：
- **总参数很大（专家多）**
- **单次计算量较小（只跑部分专家）**

---

## 三、关键优势
✅ **参数效率高**
- 模型容量大，但推理成本可控

✅ **可扩展性强**
- 增加专家数 ≠ 线性增加算力

✅ **适合大规模训练**
- 如 GPT-4、Mixtral、DeepSeek 等广泛使用

---

## 四、主要挑战
⚠ **负载不均衡**
- 某些专家被频繁调用，其他闲置
- 常用辅助损失（load balancing loss）缓解

⚠ **训练不稳定**
- 路由决策离散，优化困难

⚠ **推理与部署复杂**
- 需要支持动态路由和专家调度

⚠ **通信开销（分布式）**
- 专家常分布在不同设备，需 All-to-All 通信

---

## 五、典型代表模型
- **GShard / Switch Transformer**（Google）
- **Mixtral 8×7B**（Mistral）
- **DeepSeek-V2 / V3**
- **Qwen-MoE**
- **GPT-4（据传为 MoE）**

---

## 六、一句话总结
> **MoE 用“多个专家 + 智能路由”，让模型“大而省”，是当今大模型Scaling的重要技术路线之一。**
